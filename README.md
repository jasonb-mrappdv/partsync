# PartSync — Cloudflare edition

B2B parts-fulfillment dashboard. Originally generated by [Base44](https://base44.com); now runs entirely on Cloudflare (Pages + Functions + D1 + R2 + KV).

## Stack

| Layer | Technology |
| --- | --- |
| UI | Vite + React 18 + Tailwind + shadcn/ui |
| Routing | react-router-dom (SPA) |
| Backend API | Cloudflare Pages Functions (`/functions/api/*`) |
| Database | Cloudflare D1 (SQLite) |
| File storage | Cloudflare R2 |
| Sessions / password-reset tokens | Cloudflare KV |
| Auth | Self-hosted JWT in HttpOnly cookie (PBKDF2-SHA256 passwords) |
| Email | [Resend](https://resend.com) |

## Local development

```bash
npm install

# 1) Create local D1 + KV + R2 the first time
wrangler d1 create partsync                       # paste the id into wrangler.toml [[d1_databases]].database_id
wrangler kv namespace create SESSIONS             # paste the id into wrangler.toml [[kv_namespaces]].id
wrangler r2 bucket create partsync-uploads

# 2) Apply migrations locally
npm run db:migrate:local

# 3) Set secrets for local dev (these live in .dev.vars):
cat > .dev.vars <<'EOF'
JWT_SECRET=dev-only-change-me-to-a-long-random-string
RESEND_API_KEY=re_test_xxx
RESEND_FROM=PartSync <onboarding@resend.dev>
PUBLIC_BUCKET_URL=http://127.0.0.1:8788
APP_ENV=development
EOF

# 4) Two terminals:
#    Terminal A — Pages Functions + bindings (port 8788)
npx wrangler pages dev . --port 8788

#    Terminal B — Vite dev server (port 5173, proxies /api → 8788)
npm run dev
```

Open http://localhost:5173. Register the first user — they automatically get role `admin`.

## Deployment to Cloudflare Pages

```bash
# One-time per env: create remote D1 + KV + R2, then capture their IDs into wrangler.toml
wrangler d1 create partsync
wrangler kv namespace create SESSIONS
wrangler r2 bucket create partsync-uploads

# Apply migrations against the remote D1
npm run db:migrate:remote

# Set production secrets
wrangler pages secret put JWT_SECRET
wrangler pages secret put RESEND_API_KEY
wrangler pages secret put RESEND_FROM

# Deploy
npm run deploy
```

Connect the Pages project to GitHub via the Cloudflare dashboard for CI deploys on push to `main`. Build settings:

- Framework preset: `Vite`
- Build command: `npm run build`
- Build output directory: `dist`
- Node version: `20`

## Data migration from Base44

```bash
# 1) Grab an access token from a logged-in Base44 browser tab
#    (localStorage key `base44_access_token`) or from the Base44 dashboard.

BASE44_APP_ID=6a163d5dd3f7f69b380d7ba8 \
BASE44_APP_BASE_URL=https://partsync-flow-link.base44.app \
BASE44_ACCESS_TOKEN=eyJ... \
  node scripts/export-from-base44.mjs

# 2) Generate the import SQL
node scripts/import-to-d1.mjs

# 3) Apply
wrangler d1 execute partsync --remote --file=./data/_import.sql
```

The exporter does **not** carry over Base44 users (different auth scheme). After migrating data, register fresh accounts via `/register` (first registration becomes admin).

## Role model

Set via the `users.role` column. Values:

- `admin` — sees everything
- `vendor` — scoped to `vendor_id == user.vendor_id` for `PartOrder` and `ReturnLog`. Set `users.vendor_id` to bind a user to a vendor record.
- `technician` — scoped to their own `TechnicianLog` rows by email
- `user` — default (no admin tools)

To promote a user:

```bash
wrangler d1 execute partsync --remote --command "UPDATE users SET role='admin' WHERE email='you@example.com'"
```

## Architecture notes

- `src/api/base44Client.js` is a hand-rolled shim that mirrors the surface of `@base44/sdk` used by the pages, so the existing UI code is untouched.
- The Base44 event-triggered function `onOrderTrackingAdded` is implemented inline in `functions/api/entities/[entity]/[id].ts`: when a `PartOrder` PATCH transitions `tracking_number` from empty to set, it sends the email and flips `notification_sent`.
- The `sendTrackingEmail` manual function lives at `/api/functions/sendTrackingEmail` — a 1:1 port of the original Deno entry.
- Auth uses an HttpOnly cookie (`partsync_session`) holding a HS256 JWT signed by `JWT_SECRET`. No client-side token store.
